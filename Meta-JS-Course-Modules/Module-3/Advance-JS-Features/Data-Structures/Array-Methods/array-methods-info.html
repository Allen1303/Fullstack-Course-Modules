What are Array Methods?

Array methods are built-in JavaScript functions that operate on arrays, allowing you to transform, filter, and process data efficiently. The three most powerful methods - map(), filter(), and reduce() - form the foundation of functional programming in JavaScript, enabling you to write cleaner, more declarative code.
üîç The Big Three Overview 
const numbers = [1, 2, 3, 4, 5];

// map() - Transform each element
const doubled = numbers.map(n => n * 2);
console.log(doubled); // [2, 4, 6, 8, 10]

// filter() - Keep elements that match criteria
const evens = numbers.filter(n => n % 2 === 0);
console.log(evens); // [2, 4]

// reduce() - Combine all elements into single value
const sum = numbers.reduce((acc, n) => acc + n, 0);
console.log(sum); // 15

üó∫Ô∏è map() Method

The map() method creates a new array by transforming each element of the original array. It applies a function to every element and returns a new array of the same length with the transformed values.
üîÑ How map() Works 
// Basic transformation
const prices = [10, 25, 30, 15];
const pricesWithTax = prices.map(price => price * 1.1);
console.log(pricesWithTax); // [11, 27.5, 33, 16.5]

// Working with objects
const products = [
  { name: 'Laptop', price: 999 },
  { name: 'Phone', price: 599 }
];

const productCards = products.map(product => ({
  title: product.name.toUpperCase(),
  displayPrice: `$${product.price}`,
  affordable: product.price < 700
}));

console.log(productCards);
// [{ title: 'LAPTOP', displayPrice: '$999', affordable: false }, ...]

üîç filter() Method

The filter() method creates a new array containing only the elements that pass a test function. It's perfect for selecting subsets of data based on specific criteria, returning a new array that may be shorter than the original.
üéØ How filter() Works 
// Filter numbers
const ages = [16, 21, 18, 25, 14, 30];
const adults = ages.filter(age => age >= 18);
console.log(adults); // [21, 18, 25, 30]

// Complex filtering with objects
const employees = [
  { name: 'Alice', department: 'Engineering', salary: 75000, experience: 3 },
  { name: 'Bob', department: 'Marketing', salary: 55000, experience: 2 },
  { name: 'Charlie', department: 'Engineering', salary: 85000, experience: 5 }
];

// Multiple criteria filtering
const seniorEngineers = employees.filter(emp => 
  emp.department === 'Engineering' && 
  emp.experience >= 3 && 
  emp.salary > 70000
);

console.log(seniorEngineers); // [Alice, Charlie]

üéØ reduce() Method

The reduce() method is the most powerful array method. It processes each element to build up a single result value. Think of it as folding an array into a single value - whether that's a number, string, object, or even another array.
‚ö° How reduce() Works 
// Basic sum
const numbers = [1, 2, 3, 4, 5];
const sum = numbers.reduce((acc, num) => acc + num, 0);
console.log(sum); // 15

// Finding maximum value
const max = numbers.reduce((acc, num) => num > acc ? num : acc);
console.log(max); // 5

// Building objects from arrays
const inventory = [
  { category: 'Electronics', value: 1000 },
  { category: 'Books', value: 200 },
  { category: 'Electronics', value: 800 },
  { category: 'Clothing', value: 500 }
];

// Group by category and sum values
const categoryTotals = inventory.reduce((acc, item) => {
  acc[item.category] = (acc[item.category] || 0) + item.value;
  return acc;
}, {});

console.log(categoryTotals);
// { Electronics: 1800, Books: 200, Clothing: 500 }

// Flattening nested arrays
const nested = [[1, 2], [3, 4], [5, 6]];
const flattened = nested.reduce((acc, arr) => acc.concat(arr), []);
console.log(flattened); // [1, 2, 3, 4, 5, 6]

üîÅ forEach() Method (Quick Overview)

The forEach() method executes a function for each array element but doesn't return a new array. Use it when you need side effects (like logging, DOM manipulation, or API calls) rather than data transformation.
üîÑ forEach() Usage 
// Simple iteration
const names = ['Alice', 'Bob', 'Charlie'];
names.forEach(name => console.log(`Hello, ${name}!`));

// DOM manipulation example
const buttons = ['Save', 'Cancel', 'Delete'];
buttons.forEach(buttonText => {
  const button = document.createElement('button');
  button.textContent = buttonText;
  // document.body.appendChild(button);
});

// Note: Use map(), filter(), reduce() for data transformation
// Use forEach() only for side effects!

When to Use Each Method

Choosing the right method is crucial for writing effective JavaScript. Here's your decision-making guide for selecting the perfect array method for your specific use case:

üó∫Ô∏è Use map() when...

You need to transform every element in an array and get a new array of the same length. Perfect for data formatting, type conversion, or applying calculations.

üîç Use filter() when...

You need to select specific elements that meet certain criteria. Great for search functionality, data validation, or showing/hiding content.

üéØ Use reduce() when...

You need to combine all elements into a single value or data structure. Perfect for calculations, grouping, flattening, or building complex objects.

üîÅ Use forEach() when...

You need to perform side effects like logging, DOM updates, or API calls. Don't use it for data transformation - that's what the other methods are for!

// E-commerce product processing pipeline
const rawProducts = [
  { id: 1, name: 'laptop', price: 999, category: 'electronics', inStock: true, rating: 4.5 },
  { id: 2, name: 'mouse', price: 25, category: 'electronics', inStock: false, rating: 4.2 },
  { id: 3, name: 'book', price: 15, category: 'education', inStock: true, rating: 4.8 },

];

// Complex data processing pipeline
const result = rawProducts
  .filter(product => product.inStock && product.rating >= 4.5) // Only in-stock, highly-rated items
  .map(product => ({

How Array Methods Work

Understanding the mechanics behind these methods helps you use them more effectively. Each method follows functional programming principles: they don't modify the original array (immutability) and use pure functions for predictable results.
üîß Method Signatures & Parameters 
// map(callback(element, index, array), thisArg)
const withIndex = ['a', 'b', 'c'].map((item, index) => `${index}: ${item}`);
console.log(withIndex); // ['0: a', '1: b', '2: c']

// filter(callback(element, index, array), thisArg)
const evenIndexes = [10, 20, 30, 40].filter((item, index) => index % 2 === 0);
console.log(evenIndexes); // [10, 30] (items at even indexes)

// reduce(callback(accumulator, element, index, array), initialValue)
const indexedSum = [1, 2, 3].reduce((acc, val, idx) => {
  console.log(`Step ${idx}: ${acc} + ${val} = ${acc + val}`);
  return acc + val;
}, 0);
// Step 0: 0 + 1 = 1
// Step 1: 1 + 2 = 3
// Step 2: 3 + 3 = 6
üí° Common Patterns & Best Practices 
// 1. Always return values in map()
const numbers = [1, 2, 3];
const doubled = numbers.map(n => n * 2); // ‚úÖ Good
const undefined = numbers.map(n => { n * 2; }); // ‚ùå Missing return!

// 2. Use arrow functions for simple operations
const squares = numbers.map(n => n ** 2); // ‚úÖ Concise
const squaresVerbose = numbers.map(function(n) { return n ** 2; }); // Works but verbose

// 3. Chain methods efficiently
const data = [1, 2, 3, 4, 5, 6];
const result = data
  .filter(n => n % 2 === 0) // Filter first (reduces array size)
  .map(n => n ** 2) // Then transform
  .reduce((sum, n) => sum + n, 0); // Finally aggregate

// 4. Handle edge cases
const safeSum = (arr) => arr.length > 0 ? arr.reduce((a, b) => a + b) : 0;
const emptyArraySum = safeSum([]); // 0 instead of error

// 5. Debugging tips
const debugChain = data
  .filter(n => {
    const isEven = n % 2 === 0;
    console.log(`Filtering ${n}: ${isEven}`);
    return isEven;
  })
  .map(n => {
    const squared = n ** 2;
    console.log(`Mapping ${n} to ${squared}`);
    return squared;
  });

Performance Tips & Advanced Techniques

Master these advanced concepts to write more efficient and elegant code. Understanding performance implications and advanced patterns will make you a more effective JavaScript developer.
‚ö° Performance Optimization 
// 1. Order matters in chaining - filter first to reduce iterations
const largeArray = Array.from({ length: 1000000 }, (_, i) => i);

// ‚ùå Inefficient - maps all items then filters
const inefficient = largeArray
  .map(n => n * 2) // 1M operations
  .filter(n => n % 10 === 0); // 1M operations

// ‚úÖ Efficient - filters first, reducing subsequent operations
const efficient = largeArray
  .filter(n => (n * 2) % 10 === 0) // 1M operations
  .map(n => n * 2); // ~100K operations

// 2. Use for...of for simple iterations when you don't need a new array
let sum = 0;
for (const num of largeArray) {
  sum += num; // Faster than reduce for simple operations
}

// 3. Early returns in functions passed to array methods
const findFirstEven = numbers => {
  for (const num of numbers) {
    if (num % 2 === 0) return num; // Stops at first match
  }
};
const firstEven = numbers.find(n => n % 2 === 0); // Built-in equivalent
üéØ Advanced Patterns 
// 1. Creating reusable transformation functions
const createMultiplier = (factor) => n => n * factor;
const createFilter = (predicate) => arr => arr.filter(predicate);

const double = createMultiplier(2);
const filterEvens = createFilter(n => n % 2 === 0);

const processed = [1, 2, 3, 4].map(double); // [2, 4, 6, 8]
const evens = filterEvens(processed); // [2, 4, 6, 8]

// 2. Functional composition
const pipe = (...fns) => value => fns.reduce((acc, fn) => fn(acc), value);

const processNumbers = pipe(
  arr => arr.filter(n => n > 0), // Remove negatives
  arr => arr.map(n => n ** 2), // Square them
  arr => arr.reduce((sum, n) => sum + n, 0) // Sum them
);

const result = processNumbers([-2, 1, 3, -1, 4]); // 26 (1¬≤ + 3¬≤ + 4¬≤)

// 3. Working with complex nested data
const students = [
  { name: 'Alice', grades: [85, 92, 78], active: true },
  { name: 'Bob', grades: [76, 88, 92], active: false },
  { name: 'Charlie', grades: [95, 89, 94], active: true }
];

const activeStudentAverages = students
  .filter(student => student.active)
  .map(student => ({
    name: student.name,
    average: student.grades.reduce((sum, grade) => sum + grade, 0) / student.grades.length
  }))
  .sort((a, b) => b.average - a.average);

console.log(activeStudentAverages);
// [{ name: 'Charlie', average: 92.67 }, { name: 'Alice', average: 85 }]
