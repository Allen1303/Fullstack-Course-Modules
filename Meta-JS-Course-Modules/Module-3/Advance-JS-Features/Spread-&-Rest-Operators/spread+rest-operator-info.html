What are Spread & Rest Operators?

The spread ... and rest ... operators both use the same three-dot syntax but serve opposite purposes. The spread operator expands elements from arrays or objects, while the rest operator collects multiple elements into a single array or object. Think of spread as "unpacking" and rest as "packing."
üì§ Spread Operator - Expanding Elements 
// Spread with arrays - expands array elements
const numbers = [1, 2, 3];
const moreNumbers = [4, 5, 6];

// Old way - using concat
const combined = numbers.concat(moreNumbers);

// With spread - much cleaner!
const spreadCombined = [...numbers, ...moreNumbers];
console.log(spreadCombined); // Output: [1, 2, 3, 4, 5, 6]

// Spread with objects - merges object properties
const person = { name: 'Alice', age: 30 };
const contact = { email: 'alice@example.com', phone: '123-456-7890' };

const fullProfile = { ...person, ...contact };
console.log(fullProfile);
// Output: { name: 'Alice', age: 30, email: 'alice@example.com', phone: '123-456-7890' }
üì• Rest Operator - Collecting Elements 
// Rest with arrays - collects remaining elements
const [first, second, ...remaining] = [1, 2, 3, 4, 5];
console.log(first); // Output: 1
console.log(second); // Output: 2
console.log(remaining); // Output: [3, 4, 5]

// Rest with objects - collects remaining properties
const user = { id: 1, name: 'John', email: 'john@example.com', role: 'admin' };
const { id, ...userWithoutId } = user;
console.log(id); // Output: 1
console.log(userWithoutId); 
// Output: { name: 'John', email: 'john@example.com', role: 'admin' }

// Rest in function parameters - accepts unlimited arguments
function sum(...numbers) {
  return numbers.reduce((total, num) => total + num, 0);
}

console.log(sum(1, 2, 3, 4)); // Output: 10

üÜö Before vs After Comparison 
// OLD WAY - Verbose and harder to understand
function mergeArraysOld(arr1, arr2, arr3) {
  return arr1.concat(arr2).concat(arr3);
}

function copyObjectOld(original, updates) {
  return Object.assign({}, original, updates);
}

function getFirstAndRestOld(arr) {
  return {
    first: arr[0],
    rest: arr.slice(1)
  };
}

// NEW WAY - Clean and intuitive with spread/rest
const mergeArrays = (...arrays) => [].concat(...arrays);
// Or even simpler:
const mergeArraysSimple = (...arrays) => [...arrays.flat()];

const copyObject = (original, updates) => ({ ...original, ...updates });

const getFirstAndRest = ([first, ...rest]) => ({ first, rest });

Advanced Techniques & Patterns

Beyond basic usage, spread and rest operators enable sophisticated programming patterns. Let's explore advanced techniques that will make you a more effective JavaScript developer.
üéØ Advanced Spread Techniques 
// 1. Conditional spreading
const isAdmin = true;
const user = {
  name: 'Alice',
  email: 'alice@example.com',
  ...(isAdmin && { adminRights: ['read', 'write', 'delete'] })
};
console.log(user); // Includes adminRights only if isAdmin is true

// 2. Converting NodeLists and strings to arrays
const elements = [...document.querySelectorAll('.item')]; // NodeList to Array
const letters = [...'hello']; // Output: ['h', 'e', 'l', 'l', 'o']

// 3. Finding max/min in arrays
const scores = [89, 94, 76, 91, 88];
const highScore = Math.max(...scores); // Output: 94
const lowScore = Math.min(...scores); // Output: 76

// 4. Shallow cloning with modifications
const original = { a: 1, b: 2, c: 3 };
const modified = { ...original, b: 20, d: 4 }; // Override b, add d
console.log(modified); // Output: { a: 1, b: 20, c: 3, d: 4 }
üèóÔ∏è Advanced Rest Patterns 
// 1. Complex destructuring with rest
const data = {
  id: 1,
  name: 'Product A',
  price: 99.99,
  category: 'Electronics',
  inStock: true,
  weight: 2.5,
  dimensions: { length: 10, width: 5, height: 3 }
};

// Extract essential info, keep everything else as metadata
const { id, name, price, ...metadata } = data;
console.log(`${name}: $${price}`);
console.log('Additional info:', metadata);

// 2. Function parameter patterns
function processOrder(orderId, priority = 'normal', ...items) {
  console.log(`Processing order ${orderId} (${priority}) with ${items.length} items`);
  return items.map(item => ({ ...item, orderId, priority }));
}

const result = processOrder(123, 'urgent', 
  { product: 'Laptop', qty: 1 }, 
  { product: 'Mouse', qty: 2 }
);

// 3. Array processing with head/tail pattern
function processQueue([current, ...remaining]) {
  if (!current) return 'Queue is empty';
  
  console.log(`Processing: ${current}`);
  console.log(`Remaining: ${remaining.length} items`);
  
  return remaining.length > 0 ? processQueue(remaining) : 'Queue completed';
}

const queue = ['Task A', 'Task B', 'Task C'];
processQueue(queue);

üéõÔ∏è API & Configuration

// 1. React State Updates (immutable patterns)
const [todos, setTodos] = useState([]);

// Add todo
const addTodo = (newTodo) => {
  setTodos(prevTodos => [...prevTodos, newTodo]);
};

// Update todo
const updateTodo = (id, updates) => {
  setTodos(prevTodos => 
    prevTodos.map(todo => 
      todo.id === id ? { ...todo, ...updates } : todo
    )
  );
};
