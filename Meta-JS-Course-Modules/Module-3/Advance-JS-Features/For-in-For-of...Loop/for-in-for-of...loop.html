for...in vs for...of Loops

Master JavaScript's powerful iteration methods

Quick Overview

for...in iterates over enumerable properties (keys) of objects, while for...of iterates over iterable values like arrays and strings.

üèóÔ∏è for...in Loop

Iterates: Object properties (keys)
Best for: Objects
Returns: Property names as strings
Gotcha: Includes inherited properties

üìã for...of Loop

Iterates: Iterable values
Best for: Arrays, strings, Maps, Sets
Returns: Actual values
Gotcha: Doesn't work with plain objects

üîç Basic Comparison 
const fruits = ['apple', 'banana', 'orange'];

// for...in loops through indices (keys)
for (const index in fruits) {
  console.log(index, fruits[index]); 
  // Output: 0 apple, 1 banana, 2 orange
}

// for...of loops through values directly
for (const fruit of fruits) {
  console.log(fruit); 
  // Output: apple, banana, orange
}

üèóÔ∏è
for...in Loop

Perfect for iterating over object properties when you need both keys and values.
üîë Object Iteration 
const person = {
  name: 'Alice',
  age: 30,
  city: 'New York'
};

// ‚úÖ SAFE: Always use hasOwnProperty()
for (const key in person) {
  if (person.hasOwnProperty(key)) {
    console.log(`${key}: ${person[key]}`);
  }
}

// ‚úÖ MODERN: Use Object.hasOwn() (ES2022)
for (const key in person) {
  if (Object.hasOwn(person, key)) {
    console.log(`${key}: ${person[key]}`);
  }
}

‚ö†Ô∏è Array Gotcha

Using for...in with arrays can include custom properties. Prefer for...of for arrays unless you specifically need to check properties.

üìã
for...of Loop

The modern, clean way to iterate over arrays, strings, Maps, Sets, and other iterables.
üéØ Iterable Examples 
// Arrays - clean and simple
const numbers = [10, 20, 30];
for (const num of numbers) {
  console.log(num * 2);
}

// Strings - character by character
for (const char of 'Hello') {
  console.log(char); // H, e, l, l, o
}

// Get index + value with entries()
for (const [index, value] of numbers.entries()) {
  console.log(`${index}: ${value}`);
}

// Maps and Sets
const map = new Map([['a', 1], ['b', 2]]);
for (const [key, value] of map) {
  console.log(`${key}: ${value}`);
}

const set = new Set(['red', 'blue', 'green']);
for (const color of set) {
  console.log(color);
}

Quick Decision Guide

Choose the right loop based on your data structure and needs.
ü§î When to Use Which 
// ‚úÖ Use for...in for objects
const obj = { a: 1, b: 2 };
for (const key in obj) {
  console.log(key, obj[key]);
}

// ‚úÖ Use for...of for arrays
const arr = [1, 2, 3];
for (const value of arr) {
  console.log(value);
}

// ‚úÖ Object iteration with for...of
for (const key of Object.keys(obj)) {
  console.log(key);
}

for (const [key, value] of Object.entries(obj)) {
  console.log(key, value);
}

// ‚ùå Avoid: for...in with arrays (usually wrong)
// ‚ùå Error: for...of with plain objects

üöÄ Best Practices

‚Ä¢ Use hasOwnProperty() with for...in
‚Ä¢ Prefer for...of for arrays
‚Ä¢ Use Object.entries() for object iteration
‚Ä¢ Consider Array methods (map, filter)

‚ö†Ô∏è Common Mistakes

‚Ä¢ Using for...in with arrays
‚Ä¢ Using for...of with plain objects
‚Ä¢ Forgetting hasOwnProperty()
‚Ä¢ Not considering inherited properties

Performance Notes

Quick performance insights for optimal loop choice.
‚ö° Performance Tips 
// Fastest to slowest for arrays:
// 1. Traditional for loop (fastest)
for (let i = 0; i < arr.length; i++) { }

// 2. for...of (good performance, clean syntax)
for (const item of arr) { }

// 3. forEach (functional, readable)
arr.forEach(item => { });

// 4. for...in (avoid for arrays)
for (const index in arr) { }

// Cache array length for best performance
for (let i = 0, len = arr.length; i < len; i++) {
  // Fastest for large arrays
}
