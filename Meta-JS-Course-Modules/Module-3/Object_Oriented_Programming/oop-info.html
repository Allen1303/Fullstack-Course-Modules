What is OOP?

Object-Oriented Programming (OOP) is a programming paradigm that organizes code around objects rather than functions. It models real-world entities as objects with properties (attributes) and behaviors (methods), making code more modular, reusable, and easier to maintain.

üîí Encapsulation

Bundling data and methods together
Hide internal details and expose only necessary interfaces. Control access to object properties through methods.

üß¨ Inheritance

Creating new classes based on existing ones
Child classes inherit properties and methods from parent classes, promoting code reuse.

üé≠ Polymorphism

One interface, multiple implementations
Objects of different types can be treated uniformly through a common interface.

üé® Abstraction

Simplifying complex systems
Hide implementation details and show only essential features to reduce complexity.

üîç Basic Class Example 
// Simple class definition
class Car {
  constructor(make, model) {
    this.make = make;
    this.model = model;
    this.isRunning = false;
  }

  start() {
    this.isRunning = true;
    return `${this.make} ${this.model} started!`;
  }

  getInfo() {
    return `${this.make} ${this.model}`;
  }
}

// Create and use objects
const myCar = new Car('Toyota', 'Camry');
console.log(myCar.start()); // Toyota Camry started!

Encapsulation

Encapsulation bundles data and methods together while restricting direct access to internal components. It's like having a TV remote - you can change channels without knowing the internal circuitry.
üîê Private Fields & Controlled Access 
class BankAccount {
  // Private fields
  #balance = 0;
  
  constructor(initialBalance) {
    this.#balance = initialBalance;
  }

  deposit(amount) {
    if (amount > 0) {
      this.#balance += amount;
      return `Deposited $${amount}`;
    }
  }

  withdraw(amount) {
    if (amount <= this.#balance) {
      this.#balance -= amount;
      return `Withdrew $${amount}`;
    }
  }

  get balance() {
    return this.#balance;
  }
}

const account = new BankAccount(1000);
account.deposit(500);
console.log(account.balance); // 1500
// account.#balance = 999; // ‚ùå Can't access directly

Inheritance

Inheritance allows creating new classes based on existing ones. Child classes inherit properties and methods from parent classes while being able to add their own unique features.
üë®‚Äçüë¶ Parent and Child Classes 
// Parent class
class Animal {
  constructor(name) {
    this.name = name;
  }

  eat() {
    return `${this.name} is eating`;
  }

  makeSound() {
    return `${this.name} makes a sound`;
  }
}

// Child class
class Dog extends Animal {
  constructor(name, breed) {
    super(name); // Call parent constructor
    this.breed = breed;
  }

  // Override parent method
  makeSound() {
    return `${this.name} barks: Woof!`;
  }

  // New method
  fetch() {
    return `${this.name} fetches the ball!`;
  }
}

const dog = new Dog('Buddy', 'Golden Retriever');
console.log(dog.eat()); // Buddy is eating (inherited)
console.log(dog.makeSound()); // Buddy barks: Woof! (overridden)
console.log(dog.fetch()); // Buddy fetches the ball! (new)

üé≠
Polymorphism

Polymorphism allows objects of different types to be treated uniformly through a common interface. The same method call can produce different behaviors depending on the object type.
üîÑ One Interface, Multiple Behaviors 
class Shape {
  calculateArea() {
    throw new Error('Must be implemented');
  }
}

class Circle extends Shape {
  constructor(radius) {
    super();
    this.radius = radius;
  }

  calculateArea() {
    return Math.PI * this.radius ** 2;
  }
}

class Rectangle extends Shape {
  constructor(width, height) {
    super();
    this.width = width;
    this.height = height;
  }

  calculateArea() {
    return this.width * this.height;
  }
}

// Polymorphism in action
const shapes = [
  new Circle(5),
  new Rectangle(4, 6)
];

shapes.forEach(shape => {
  console.log(`Area: ${shape.calculateArea()}`);
});

Abstraction

Abstraction hides complex implementation details and exposes only the essential features. It's like driving a car - you don't need to understand the engine internals to operate the vehicle.
üéØ Abstract Classes & Hidden Complexity 
// Abstract base class
class Database {
  constructor() {
    if (this.constructor === Database) {
      throw new Error('Cannot instantiate abstract class');
    }
  }

  connect() { throw new Error('Must implement'); }
  query(sql) { throw new Error('Must implement'); }
}

// Concrete implementation
class MySQL extends Database {
  connect() {
    // Complex connection logic hidden
    return 'Connected to MySQL';
  }

  query(sql) {
    // Complex query processing hidden
    return `Executing: ${sql}`;
  }
}

// High-level service uses simple interface
class UserService {
  constructor(db) {
    this.db = db;
  }

  getUsers() {
    this.db.connect();
    return this.db.query('SELECT * FROM users');
  }
}

const service = new UserService(new MySQL());
console.log(service.getUsers());

OOP Best Practices

üéØ Single Responsibility

Each class should have one reason to change. Keep classes focused on a single task.

üîí Favor Composition

Prefer object composition over inheritance when possible for more flexibility.

üìã Program to Interfaces

Depend on abstractions, not concrete implementations for better maintainability.

üîÑ Open/Closed Principle

Classes should be open for extension but closed for modification.

‚ö†Ô∏è Common Pitfalls to Avoid

‚Ä¢ God Objects: Classes that do too much
‚Ä¢ Deep Inheritance: Avoid chains more than 3-4 levels deep
‚Ä¢ Breaking Encapsulation: Don't expose internal state unnecessarily
‚Ä¢ Tight Coupling: Classes shouldn't depend heavily on each other's internals

Summary

Object-Oriented Programming provides a powerful way to organize and structure code by modeling real-world concepts as objects. The four core principles work together to create maintainable, scalable, and reusable software systems.

üîí Encapsulation

Bundle data and methods, control access
Use private fields and getters/setters to protect data integrity.

üß¨ Inheritance

Extend existing classes, promote reuse
Create specialized classes that inherit base functionality.

üé≠ Polymorphism

Same interface, different behaviors
Write flexible code that works with multiple object types.

üé® Abstraction

Hide complexity, expose essentials
Create clear interfaces that hide implementation details.
